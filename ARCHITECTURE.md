# ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ МОДА: RUDAZOV MOD

## 1. Технический стек
* Версия Minecraft: 1.12.2
* Загрузчик модов: Minecraft Forge (FML / EventBus API)
* Версия Java: Java 8
* Mod ID: rudazovmod
* Базовый пакет: com.poleesteel.rudazovmod

## 2. Зарегистрированные предметы, блоки и сущности
* Предметы:
    * `ItemBloodChain` (ID: "blood_chain", вкладка CreativeTabs.COMBAT, макс. стак: 1, прочность: 250). Оружие ближнего боя и инструмент дальнего контроля.
    * Базовые предметы (слиток кровавого железа, руда, блоки) — генерация в мире временно отложена.
* Сущности:
    * `EntityBloodChain` (ID: "blood_chain_entity", Network ID: 1, tracker: range 64, update 1, sendVelocity true). Невидимая физическая сущность, отвечающая за логику контроля моба и точки привязки рендера.
* Рендереры:
    * `RenderBloodChain` — клиентский процедурный OpenGL-рендерер для `EntityBloodChain`.

## 3. Реализованная логика и методы
* Класс `ItemBloodChain`:
    * `getItemAttributeModifiers`: Задает характеристики для MAINHAND (+6.0D к урону, -2.4D скорость атаки).
    * `itemInteractionForEntity`: Срабатывает при ПКМ из любой руки (Main или Off-hand). Защищен от спама через `hasCooldown()`. Реализует ограничение "1 игрок = 1 цель": при активации ищет в мире и удаляет (`setDead`) все старые `EntityBloodChain`, где игрок является хозяином. Спавнит новую сущность на сервере, издает звук `ITEM_ARMOR_EQUIP_CHAIN`, накладывает кулдаун в 60 тиков (3 сек).
    * `hitEntity`: Ванильная melee-атака с расходом прочности (`damageItem`).
* Класс `EntityBloodChain`:
    * Конструктор: Установлен параметр `noClip = true` (отключение коллизии с блоками/мобами для предотвращения бесконечного цикла расчетов физики и зависания сервера). Спавн происходит строго по координатам цели через `setLocationAndAngles` (решение проблемы загрузки нулевого чанка X:0, Z:0).
    * Синхронизация: Использует `DataParameter<Integer>` для передачи `TARGET_ID` (жертва) и `OWNER_ID` (игрок) с сервера на клиент.
    * `onUpdate` (Логика контроля): Привязывает свои координаты к центру хитбокса цели. Разрывает цепь (`setDead`), если хозяин умер или отдалился более чем на 20 блоков.
    * Абсолютный паралич цели: Обнуляет векторы движения (`motionX = 0`, `motionZ = 0`), гасит прыжки (`motionY = -0.1D` для легкой гравитации). Накладывает эффект `SLOWNESS` 10-го уровня на 60 тиков (отключение ИИ перемещения). Обновляет зелье только при остатке длительности <= 20 тиков, флаг `showParticles = false` (предотвращает сетевой DDoS-спам пакетами зелий).
* Класс `RenderBloodChain` (OpenGL / Tessellator):
    * `shouldRender`: Всегда возвращает `true` (отключен ванильный Frustum Culling, цепь не исчезает при повороте камеры или нахождении моба за спиной/на краю экрана).
    * Отрисовка 1 (Поводок): Процедурная линия из 3D-звеньев от центра сущности до груди/рук игрока (`owner.height * 0.5D`). Использует правостороннюю систему координат Minecraft (`atan2(dx, dz)`) для точного прицеливания.
    * Отрисовка 2 (Спираль): Процедурная 3D-цепь вокруг хитбокса моба (радиус рассчитывается по диагонали моба, высота — по высоте моба, 45 сегментов, 2.5 витка, динамическое вращение со временем).
    * Геометрия: Звенья рисуются через `GL_QUADS`. Каждое четное звено повернуто на 90° по оси Z (эффект переплетения). Добавлен нахлест (Z-offset). Освещение отключено (`disableLighting`), включено альфа-смешивание для визуального свечения текстуры `rudazovmod:textures/entity/blood_chain_link.png`.

## 4. Особенности архитектуры
* Регистрация (`RegistryHandler` в пакете `init`): Все регистрации проходят через ванильные события Forge (`@SubscribeEvent`). Метод `onItemRegister` (регистрация предметов), `onEntityRegister` (`EntityEntryBuilder`), `onModelRegister` с аннотацией `@SideOnly(Side.CLIENT)` (регистрация моделей и рендереров сущностей через `RenderingRegistry.registerEntityRenderingHandler`).
* Разделение логики: Жесткое разделение клиентского визуального кода (пакет `client.render.entity`) и серверной физики (пакет `entities`). Механика оружия и магии объединена в одном классе предмета без создания лишних обработчиков событий (Events), что экономит ресурсы сервера.
